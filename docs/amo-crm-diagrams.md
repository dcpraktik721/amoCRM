# Mermaid диаграммы реализации amoCRM для клиники Практик

Документ содержит рабочие Mermaid-схемы, которые иллюстрируют архитектуру интеграций, путь пациента, пять воронок amoCRM и роли. Все схемы рендерятся в любом просмотрщике Markdown с поддержкой Mermaid (GitHub, Obsidian, VS Code и т.д.).

---

## 1. Общая архитектура интеграций

```mermaid
flowchart LR
    subgraph SRC["Источники заявок"]
      direction TB
      S1["Яндекс Директ\nпоиск, РСЯ, ретаргетинг"]
      S2["VK Реклама,\nVK сообщество, VK органика"]
      S3["Геоагрегаторы\nЯндекс Карты, 2ГИС"]
      S4["Медагрегаторы\nНапоправку, Продокторов"]
      S5["SEO\nсайт, блог, услуги"]
      S6["Прямые\nзвонок, бренд, сайт"]
      S7["Рекомендации"]
      S8["Офлайн\nвывеска, листовка"]
      S9["Повторные пациенты"]
      S10["Реактивация базы"]
    end

    SRC -->|звонок, форма, чат| CT["Calltouch\nUTM, источник, запись"]
    CT -->|обращение, UTM| AL["Альбато\nOAuth, маппинг"]
    AL -->|сделка, контакт| AMO["amoCRM\nворонки, задачи, SLA"]

    DP["DentalPRO\nрасписание, медкарта,\nплан, оплаты, факт визита"] -->|webhook статусов| AMO
    AMO -->|статус сделки| AL
    AL -->|обратная аналитика| CT

    AMO --> R1["Руководитель"]
    AMO --> R2["Координатор лечения"]
    AMO --> R3["Маркетолог"]
    AMO --> R4["Администратор"]

    classDef amo fill:#e8f1fc,stroke:#1e6fd9,color:#1d2733;
    classDef dental fill:#e5f5ec,stroke:#1f8f5a,color:#1d2733;
    classDef calltouch fill:#efe7f7,stroke:#7a3fb6,color:#1d2733;
    classDef albato fill:#fdf0e0,stroke:#e07b1a,color:#1d2733;
    classDef src fill:#f4f6f9,stroke:#cfd6df,color:#1d2733;

    class AMO amo;
    class DP dental;
    class CT calltouch;
    class AL albato;
    class S1,S2,S3,S4,S5,S6,S7,S8,S9,S10,SRC src;
```

## 8. Поток Calltouch -> Альбато -> amoCRM

```mermaid
sequenceDiagram
    participant P as Пациент
    participant CT as Calltouch
    participant AL as Альбато
    participant AMO as amoCRM
    participant ADM as Администратор

    P->>CT: Звонок, форма, чат
    CT->>CT: Определение источника, UTM, запись звонка
    CT->>AL: Передача обращения, UTM, ID обращения
    AL->>AL: Маппинг полей, проверка на дубль
    AL->>AMO: Создание контакта и сделки
    AMO->>AMO: Воронка 1, задача администратору, SLA таймер
    AMO->>ADM: Уведомление о новой заявке
    ADM->>P: Первый контакт до 5 минут
    AMO-->>AL: Изменение статуса сделки
    AL-->>CT: Запись статуса для сквозной аналитики
```

## 9. Поток DentalPRO -> amoCRM

```mermaid
sequenceDiagram
    participant DP as DentalPRO
    participant AMO as amoCRM
    participant CL as Координатор лечения

    DP->>AMO: webhook Запись создана\nID записи, врач, дата, филиал
    AMO->>AMO: Сделка в Записан
    DP->>AMO: webhook Подтверждение приема
    AMO->>AMO: Сделка в Подтвержден
    DP->>AMO: webhook Факт прихода
    AMO->>AMO: Сделка в Пришел и сделка в Воронке 2 План
    DP->>AMO: webhook Неявка
    AMO->>AMO: Сделка в Не пришел, требуется причина
    DP->>AMO: webhook Перенос
    AMO->>AMO: Сделка в Перенос, требуются причина и дата
    DP->>AMO: webhook План составлен сумма направление
    AMO->>AMO: Сделка в План составлен
    DP->>AMO: webhook План согласован сумма дата старта
    AMO->>AMO: Сделка в Согласован, переход в Воронку 3
    DP->>AMO: webhook Лечение начато
    AMO->>AMO: Сделка в Начал лечение
    DP->>AMO: webhook Лечение завершено
    AMO->>AMO: Сделка в Завершено и сделка в Воронке 4
    AMO->>CL: Эскалация при отсутствии активности 14+ дней
```

## 10. Карта ролей и ответственности

```mermaid
flowchart LR
    subgraph Управление
      R["Руководитель клиники"]
    end
    subgraph Удержание
      C["Координатор лечения"]
    end
    subgraph Операционка
      A["Администратор"]
    end
    subgraph Маркетинг
      M["Маркетолог"]
    end
    subgraph Медицина
      D["Врач"]
    end
    subgraph Финансы
      F["Финансист"]
    end
    subgraph Эксплуатация
      T["Технический интегратор"]
    end

    A -->|обработка заявок| AMO[(amoCRM)]
    A -->|запись| DP[(DentalPRO)]
    C -->|контроль удержания| AMO
    M -->|каналы и UTM| AMO
    M --> CT[(Calltouch)]
    R -->|отчеты| AMO
    R --> DP
    D -->|медицина| DP
    F -->|оплаты| DP
    T -->|webhooks, поля| AMO
    T --> CT
    T --> AL[(Альбато)]
    T --> DP
```
